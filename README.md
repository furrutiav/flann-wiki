# flann-wiki

## Further Steps

- [ ] Define good-practice guidelines for adding papers.
  Decide what makes a paper eligible, which fields are required, and how entries
  should be checked before merging. For example: prefer theory-oriented papers,
  verify that source URLs match the title, avoid duplicates, copy the original
  abstract when available, and write a short summary explaining why the paper
  belongs in the database.

- [ ] Define a taxonomy of concepts for cataloging papers.
  Organize the database around reusable concepts instead of one-off tags. For
  example: model families such as transformers, RNNs, state-space models;
  theoretical lenses such as expressivity, learnability, interpretability; and
  formal tools such as automata, formal languages, circuits, logic, complexity,
  or dynamical systems.

- [ ] Extract structured knowledge from the papers.
  Go beyond metadata by collecting the main definitions, methods, assumptions,
  and theoretical results from each paper. For example: definitions of model
  classes, formal languages used as benchmarks, theorem statements, proof ideas,
  separation results, lower bounds, or extraction algorithms.
